Connecter THE PILOT aux données Seven At Home

Ce document résume notre besoin technique pour brancher THE PILOT (l'outil de pilotage marketing & closing de SAH) sur vos données. Objectif : caler ensemble comment récupérer les informations, et valider ce qui est faisable de votre côté.

1 Le concept en 30 secondes

THE PILOT est un outil interne construit pour Seven At Home. Il centralise le suivi des investisseurs, le travail des closers (priorisation des appels, briefs, relances) et le reporting de performance, avec une couche d'intelligence artificielle.

Point clé : THE PILOT ne crée aucune donnée. C'est un miroir en lecture seule de votre plateforme. Toutes les données viennent de chez vous ; nous les affichons, les analysons et les croisons, mais la source de vérité reste Seven At Home.

2 Notre objectif : mesurer la rentabilité de chaque action

Le cœur du projet est de pouvoir relier nos actions (appel, SMS, WhatsApp, email, relance) aux résultats (un investisseur qui souscrit). Exemple : on appelle un inscrit qui n'a jamais investi, et 7 jours plus tard il investit 10 000 € — on veut pouvoir attribuer cet investissement à l'appel.

Pour ça, il nous faut une chose essentielle de votre côté : l'événement « souscription » avec sa date/heure précise et l'identifiant de l'investisseur concerné. C'est la clé de voûte de tout le système.

🔒
Ce que nous ne voulons PAS récupérer
Aucune donnée KYC ultra-sensible : pas de numéro de pièce d'identité, pas de RIB/IBAN, pas de scan de document, pas de mot de passe. Ces données restent chez vous (vous êtes responsables du KYC sous cadre ACPR). De notre côté, moins on stocke de données sensibles, plus l'ensemble est sûr. Le booléen « onboarding validé : oui/non » nous suffit largement.

4 Comment : les options techniques

Plusieurs méthodes sont possibles. Dites-nous laquelle est la plus simple de votre côté — on s'adapte.

Option A — Webhooks + API REST notre préférence

Vous nous « poussez » les événements en temps réel (nouvelle inscription, souscription…) via des webhooks, et on peut interroger une API REST pour les requêtes ponctuelles et le chargement initial. C'est le plus réactif et le plus propre.

Option B — Exports CSV programmés

Vous déposez régulièrement (ex. toutes les heures / chaque nuit) des fichiers CSV ou JSON sur un espace sécurisé (S3 EU ou SFTP). Plus simple à mettre en place, mais en différé.

Option C — Accès lecture seule à une réplique de base

Un accès en lecture seule à une copie (replica) de votre base de données. Très efficace, mais plus engageant côté accès — uniquement si vous êtes à l'aise avec ça.

Pour démarrer, il nous faudra aussi un export initial de l'existant (investisseurs déjà inscrits, projets, souscriptions historiques) pour amorcer la base.

5 Sécurité & conformité

  • Hébergement 100 % Union Européenne (Francfort). Aucune donnée ne sort de l'UE.
  • Authentification : nous nous connectons à votre API via un compte technique dédié (OAuth 2.0 ou clé API + secret), stocké chiffré, jamais en clair.
  • Webhooks signés (HMAC) pour qu'on vérifie que les données viennent bien de vous.
  • Chiffrement en transit (TLS) et au repos (AES-256) sur toute la chaîne.
  • RGPD : nous agissons comme sous-traitant de SAH. Un accord de traitement des données (DPA) sera signé entre nos structures. Le droit à l'oubli est propagé (si un investisseur demande l'effacement chez vous, on supprime chez nous).
  • Possibilité de restreindre l'accès à nos adresses IP de production (whitelist) si vous le souhaitez.
