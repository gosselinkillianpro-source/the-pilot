01 / Signature

Le background horizon

Plutôt que des photos qui poseraient des problèmes de lisibilité, de cohérence brand et de charge cognitive, on utilise un horizon abstrait en gradients + SVG. Tout en CSS. Performant, contrôlable, lisible.

La composition

  • Sky gradient vertical : zenith pâle (#F0F4FD) → horizon rosé (#FFE9F0) → soleil couchant (#FFDDC2) → sol (#E8ECF5)
  • Sun glow : un halo doré au milieu de l'horizon (600px, blur 20px) qui pulse doucement en 8s
  • Mountain silhouettes SVG : trois couches de montagnes en parallax visuel, des plus pâles aux plus denses
  • Un blob bleu discret en haut à droite, qui dérive lentement
  • Noise texture en overlay subtle, blend mode overlay
Pourquoi pas une photo réelle : une photo a des zones très contrastées (ciel clair, montagnes sombres) qui rendent le texte alternativement très lisible et illisible. Mes blobs colorés et mes gradients sont calibrés pour rester sous une plage de luminosité. Et une photo "raconte" un endroit, ce qui peut entrer en conflit avec ton contenu marketing. Cela dit, un mode "photo paysage" est dispo en bas à droite si tu veux comparer.

02 / Vision

Cinq principes

L'esprit v4. Moins de chaos visuel, plus de respiration.

1. Horizon apaisant en arrière-plan

Le fond raconte une métaphore : tu pilotes, l'horizon est devant toi. Calm sky + sun glow + mountains silhouettes. Couleurs douces, mouvement très lent. Le fond ne distrait jamais, il pose une ambiance.

2. Glass intentionnel sur les éléments flottants

Sidebar, topbar, modals, cards principales. Le contenu primaire (tables, formulaires denses) reste opaque pour la lisibilité. Three intensités : subtle / default / strong.

3. Respiration généreuse

Padding plus larges qu'en v3. Margins plus généreuses entre sections. Cards en radius 20-28px (plus doux, plus premium). Less items per row pour aérer.

4. Glow rare et précieux

Quand quelque chose glow dans v4, ça veut dire quelque chose. Action principale, hot lead, IA active, danger flag. Tout le reste est calme. Le glow est un signal, jamais une déco.

5. Typographie expressive sur les moments forts

Inter pour la masse, JetBrains Mono pour les chiffres tabulaires, Instrument Serif italique pour les page titles, KPI values, et hero text. Le contraste entre les fontes structure la hiérarchie sans avoir besoin de gros poids.

Pour Claude Code : v4 est plus stricte que v3 sur l'épuration. Si tu hésites à ajouter quelque chose, ne l'ajoute pas. La densité d'information reste maximale (c'est une app de power-users), mais la densité d'éléments visuels est minimale. Le fond porte la magie, le contenu fait le travail.
03 / Fondation

Liquid Glass

Adapté pour laisser passer les couleurs de l'horizon. White avec transparence (40-82%), blur fort (18-48px), saturate boost (180-220%).

Trois intensités

Subtle
Pour les très gros containers de section
Default
Pour les cards, KPIs, panels
Strong
Pour les modals, command palette
/* Subtle — laisse passer beaucoup l'horizon */ backdrop-filter: blur(18px) saturate(180%); background: rgba(255, 255, 255, 0.4); /* Default — équilibre lisibilité/transparence */ backdrop-filter: blur(28px) saturate(200%); background: rgba(255, 255, 255, 0.82); /* Strong — opaque pour modals critiques */ backdrop-filter: blur(48px) saturate(220%); background: #FFFFFF;
Performance : backdrop-filter coûte cher. Pas sur des éléments scrollés à 60fps. Pour des listes longues (50+ rows), garder du surface opaque blanc.

Setup tokens Tailwind v4

/* app/globals.css */ @import "tailwindcss"; @theme { --color-bg: #FAFBFD; --color-surface: #FFFFFF; --color-glass-bg: rgba(255, 255, 255, 0.82); --color-glass-border: rgba(255, 255, 255, 0.7); --color-brand: #2563EB; --color-ai: #7C3AED; --color-success: #059669; --color-warning: #D97706; --color-danger: #DC2626; --color-sky-top: #F0F4FD; --color-sky-mid: #FFE9F0; --color-sky-horizon: #FFDDC2; --color-ground: #E8ECF5; --color-mountain: #B8C2D9; --color-mountain-deep: #8E9BBD; --font-sans: 'Inter', system-ui, sans-serif; --font-mono: 'JetBrains Mono', monospace; --font-display: 'Instrument Serif', Georgia, serif; }

Composant HorizonBg

// app/components/horizon-bg.tsx export function HorizonBg() { return ( <> <div className="horizon-bg" /> <div className="horizon-sun" /> <div className="horizon-blob" /> <div className="horizon-mountains"> <svg viewBox="0 0 1920 400" preserveAspectRatio="none"> {/* 3 paths de montagnes */} </svg> </div> <div className="noise-overlay" /> </> ); }

Règles d'implémentation

  • HorizonBg mounted une fois dans le RootLayout. Présent sur toutes les pages.
  • Mode light en défaut. Dark mode = toggle utilisateur.
  • Glass partout sauf tables, listes longues, formulaires denses.
  • Inter par défaut, Mono pour tabular-nums, Instrument Serif italic UNIQUEMENT sur page titles, KPI values, AI script.
  • Glow conditionnel uniquement : primary button, hot lead, AI active, danger flag.
  • Tester perf des backdrop-filter sur scroll. Dégrader vers opaque si lag.
